Create `src/components/MindRadar.astro` — a build-time Astro component that renders a static inline SVG radar chart for a given country code. This is per user decision: "build-time Astro component using template literals for basic radar polygon math (~150x150px). No external SVG library."

**Props interface:**
```typescript
interface Props {
  country: string; // ISO3 code e.g. "DNK"
  size?: number;   // px, default 150
}
```

**Implementation:**
1. In the frontmatter, import `scores` from `../../data/mind-scores.json`
2. Look up `scores.countries[Astro.props.country]` to get the country's dimension data
3. Extract M/I/N/D scores from `dimensions.m.score`, `dimensions.i.score`, etc.
4. Use dimension colors matching the dashboard palette: M=#00ff88, I=#00c8ff, N=#7b4bff, D=#ffb400
5. Compute SVG polygon points for a 4-axis radar chart:
   - Center at (size/2, size/2), max radius = size/2 - 15 (padding for labels)
   - 4 axes at 0deg (top=Material), 90deg (right=Intelligence), 180deg (bottom=Network), 270deg (left=Diversity)
   - For each dimension, point = center + (score/100 * maxRadius) in that direction
   - Use standard radar math: x = cx + r * sin(angle), y = cy - r * cos(angle) for angles [0, PI/2, PI, 3PI/2]
6. Render an `<svg>` element with:
   - `viewBox="0 0 {size} {size}"` and `width="{size}" height="{size}"`
   - `class="mind-radar-inline"` and `role="img"` and `aria-label="MIND radar chart for {countryName}"`
   - 3 concentric guide circles (at 33%, 66%, 100% radius) in rgba(255,255,255,0.1) stroke, no fill
   - 4 axis lines from center to edge in rgba(255,255,255,0.15)
   - 4 small axis labels (M, I, N, D) at the outer edge of each axis in the dimension's color, font-size 10px
   - A filled polygon connecting the 4 data points with fill opacity 0.25 and stroke width 1.5, using a gradient or single color (#00ff88 at 0.25 opacity for fill, full for stroke)
   - 4 small circles (r=3) at each data point in the dimension's color
7. If country code is not found in the data, render a small `<span>` with "[No data for {country}]"
8. The component must produce ZERO client-side JavaScript — purely static SVG inline in HTML

**Style notes (per Claude's discretion on SVG visual style):**
- Use a dark background fill on the SVG: `fill="rgba(10, 10, 18, 0.6)"` as the chart background rect (matches site bg)
- Keep it compact at 150x150px default — it sits inline next to prose text
- Use `display: inline-block; vertical-align: middle;` on a wrapper span so it flows with text or floats beside paragraphs
