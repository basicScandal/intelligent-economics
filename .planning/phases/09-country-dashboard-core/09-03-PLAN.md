    **src/scripts/dashboard/state.ts** — Simple state management for dashboard selections:

    ```typescript
    import type { SlimCountry } from './search';

    export interface DashboardState {
      primary: SlimCountry | null;       // Selected country for radar + detail view
      comparison: SlimCountry[];          // Up to 4 countries for bar chart comparison (includes primary)
      isMobile: boolean;                  // Tracks viewport < 768px
    }

    type Listener = (state: DashboardState) => void;

    export function createDashboardState() {
      let state: DashboardState = { primary: null, comparison: [], isMobile: false };
      const listeners: Listener[] = [];

      return {
        get: () => state,
        selectPrimary(country: SlimCountry) { ... },    // Sets primary, adds to comparison if not present (max 4)
        addToComparison(country: SlimCountry) { ... },   // Adds to comparison array (max 4), no-op if already in
        removeFromComparison(code: string) { ... },      // Removes from comparison by code
        setMobile(isMobile: boolean) { ... },            // Updates mobile flag
        subscribe(fn: Listener) { listeners.push(fn); }, // Register state change callback
        notify() { listeners.forEach(fn => fn(state)); }, // Trigger all listeners
      };
    }
    ```

    Rules:
    - `selectPrimary`: sets primary, auto-adds to comparison if not already there (max 4). If comparison already has 4 and country isn't in it, replace the last non-primary entry. Always calls `notify()`.
    - `addToComparison`: appends if comparison.length < 4 and country not already present. Calls `notify()`.
    - `removeFromComparison`: filters out by code. If removed country was primary, set primary to null. Calls `notify()`.
    - `setMobile`: updates isMobile, calls `notify()` only if value changed.

    **Update src/pages/dashboard.astro:**

    Add the island component to the page. The island should be placed near the top of the page (after the header) but uses `client:visible` indirectly through Astro's script loading:

    1. Import the island: `import DashboardIsland from '../components/dashboard/DashboardIsland.astro';`

    2. Place `<DashboardIsland />` right after the header section but before the static content sections. Since the script in the island is a standard Astro `<script>` tag (not `is:inline`), Astro will bundle it and load it as a module. The script executes after DOM is ready.

    Note: Astro's `<script>` tags in components are automatically bundled and loaded as `type="module"`. They execute after DOM parsing but not lazily. For true `client:visible` behavior, the script should use an IntersectionObserver pattern or we can use the Astro island approach differently.

    **Alternative approach (preferred):** Since we want `client:visible` loading per the locked decision, and our island is pure vanilla TS (not a framework component), use this pattern:

    In `dashboard.astro`, add a `<script>` tag that dynamically imports init.ts when the dashboard enters the viewport:

    ```html
    <script>
      const observer = new IntersectionObserver((entries) => {
        if (entries[0].isIntersecting) {
          import('../scripts/dashboard/init');
          observer.disconnect();
        }
      }, { rootMargin: '200px' });
      const target = document.getElementById('dashboard-data');
      if (target) observer.observe(target);
    </script>
    ```

    This achieves `client:visible` semantics for a vanilla TS island — the ECharts bundle only downloads when the dashboard area enters the viewport (with 200px rootMargin for early start). Place this script at the bottom of the page template.

    If the DashboardIsland.astro component approach is cleaner for organization, put the IntersectionObserver script inside the island component file.

    **Also add a search input placeholder** that exists in static HTML (visible before JS) and gets enhanced by init.ts:
    - In the `#search-container` div, add a static `<input type="search" placeholder="Search 217 countries..." disabled />` with a `<noscript>` message
