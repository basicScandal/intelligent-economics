**src/styles/global.css** — Tailwind v4 CSS-first configuration with ALL design tokens from the existing index.html palette. Use the `@theme` directive for Tailwind-consumable tokens.

Structure:
1. `@import "tailwindcss";`
2. `@theme { ... }` block with Tailwind-consumable tokens for colors, fonts, radii, content widths
3. `:root { ... }` block with the FULL dark-theme palette from index.html lines 64-89 (all color, shadow, transition, space, text, radius, content tokens)
4. `[data-theme="light"] { ... }` block with the FULL light-theme overrides from index.html lines 92-112

For the `@theme` block, map CSS custom properties to Tailwind token names:
```css
@theme {
  --font-display: 'Clash Display', 'Helvetica Neue', sans-serif;
  --font-body: 'Satoshi', 'Inter', sans-serif;
  --color-bg: var(--color-bg);
  --color-surface: var(--color-surface);
  --color-surface-2: var(--color-surface-2);
  --color-text: var(--color-text);
  --color-text-muted: var(--color-text-muted);
  --color-primary: var(--color-primary);
  --color-primary-hover: var(--color-primary-hover);
  --color-secondary: var(--color-secondary);
  --color-tertiary: var(--color-tertiary);
  --color-border: var(--color-border);
}
```

This allows Tailwind utility classes like `bg-bg`, `text-primary`, `font-display`, `text-text-muted` to work.
