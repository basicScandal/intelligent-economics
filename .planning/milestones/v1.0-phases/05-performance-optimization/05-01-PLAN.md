**device-detect.ts changes (ARCH-05):**

The existing tier logic is already close but needs refinement. Update `getDeviceCapability()`:

1. Keep the existing `DeviceCapability` interface — no changes needed (it already has `tier: 'full' | 'reduced' | 'minimal'`).

2. Refine tier classification logic. Replace the current tier assignment block with:
   - `minimal`: if `prefersReducedMotion` is true OR `supportsWebGL` is false (keep as-is)
   - `reduced`: if `isMobile` is true (any mobile, regardless of cores) OR if desktop but `hardwareConcurrency <= 2`
   - `full`: everything else (desktop with >2 cores)

   The current logic checks `cores <= 4 && isMobile` for `isLowEnd` and then uses `isLowEnd` for `reduced`. Change: ALL mobile devices should be `reduced` (even flagship phones — the screen is small and battery matters). Desktop with very few cores (<=2, e.g. old Chromebooks) should also be `reduced`.

3. Add a `pixelRatioLimit` field to `DeviceCapability`:
   - `full`: `Math.min(window.devicePixelRatio, 2)`
   - `reduced`: `Math.min(window.devicePixelRatio, 1.5)`
   - `minimal`: 1

   This lets consumers clamp pixel ratio without re-computing tier logic.

4. Cache the result in a module-level variable so repeated calls within the same page don't re-detect. Use `let _cached: DeviceCapability | null = null;` at module scope, return cache if set.

**hero-particles.ts changes (ARCH-06, PERF-02):**

1. Replace the `isMobile ? 1500 : 4000` pattern with tier-based lookup:
   ```typescript
   const PARTICLE_COUNTS = { full: 4000, reduced: 1500, minimal: 0 } as const;
   const PARTICLE_COUNT = PARTICLE_COUNTS[device.tier];
   if (PARTICLE_COUNT === 0) return; // minimal tier — CSS fallback
   ```

2. Remove the separate `if (device.prefersReducedMotion) return;` early bail — it's now handled by tier === 'minimal' check above (since minimal is set when prefersReducedMotion is true).

3. Replace `renderer.setPixelRatio(Math.min(window.devicePixelRatio, isMobile ? 1.5 : 2))` with `renderer.setPixelRatio(device.pixelRatioLimit)`.

4. Replace remaining `isMobile` references: `antialias: device.tier === 'full'` instead of `antialias: !isMobile`.

5. Keep all other logic unchanged (nuclei positions, velocities, animation loop, mouse parallax, resize handler).
