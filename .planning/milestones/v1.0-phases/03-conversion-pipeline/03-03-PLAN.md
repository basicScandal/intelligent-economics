    **Create directory:** `netlify/functions/` (mkdir -p if needed)

    **Create `netlify/functions/submission-created.ts`:**

    This function fires automatically on every Netlify Forms submission (the function name `submission-created` is a Netlify convention — no configuration needed).

    ```typescript
    // Netlify submission-created event handler
    // Fires automatically when any Netlify Form receives a submission
    // Syncs subscriber to MailerLite for welcome email automation
    ```

    1. Export a default async handler function that receives a `Request` object (Netlify Functions v2 format)
    2. Parse the request body as JSON to get `{ payload }`
    3. Extract `payload.data.email` — if missing or empty, return 200 (no email = nothing to sync, but don't error)
    4. Extract `payload.form_name` to determine which form submitted
    5. Build the MailerLite API request body:
       - `email`: from payload.data.email
       - `fields`: For "volunteer-signup" form, include `{ name: payload.data.first_name, last_name: payload.data.last_name }`. For "email-capture" form, fields can be empty `{}`
       - `groups`: `[process.env.MAILERLITE_GROUP_ID]` — only include if env var is set
    6. POST to `https://connect.mailerlite.com/api/subscribers` with:
       - Header `Authorization: Bearer ${process.env.MAILERLITE_API_KEY}`
       - Header `Content-Type: application/json`
       - JSON stringified body
    7. If MAILERLITE_API_KEY is not set, log a warning and return 200 (graceful degradation — forms still work without email integration)
    8. If the MailerLite API call fails (network error or non-2xx response), log the error but still return 200 to Netlify. The form submission is already saved — we do NOT want to break the form flow because of an email API issue.
    9. On success, return 200 with a JSON body `{ statusCode: 200, body: "Subscriber synced" }`

    **Important:** Use Netlify Functions v2 format (export default async handler, NOT the old exports.handler pattern). The function receives a standard Request object.

    **Update `netlify.toml`:**
    Add a `[functions]` section to declare the functions directory:
    ```toml
    [functions]
      directory = "netlify/functions"
      node_bundler = "esbuild"
    ```
    Place this AFTER the `[build.environment]` section and BEFORE the redirects. Keep all existing config intact.
