    Create `src/scripts/form-handler.ts` that handles BOTH forms via a shared pattern:

    **initVolunteerForm() function:**
    1. querySelector `#volunteer-form` — bail if not found
    2. Add `submit` event listener with `e.preventDefault()`
    3. Basic client validation: email field must contain `@`, required fields checked
    4. Disable submit button, change text to "Submitting..."
    5. Collect FormData from the form element
    6. POST via fetch to the current page URL (`window.location.pathname`) with the FormData body (this is how Netlify Forms works with AJAX — POST to the same page with form-encoded data)
    7. On success (response.ok): hide the form element (`form.style.display = 'none'`), add `.visible` class to `#signup-success`, scrollIntoView smooth — per CONV-02
    8. On failure OR catch: show an error message div (see below), re-enable button with original text, provide retry — per CONV-03
    9. Error state: create/show a `.form-error` div with text "Something went wrong. Please try again." and style it with inline red border. The div should already exist in markup (Task 2 adds it). Toggle `.visible` class on it.

    **initEmailCapture() function:**
    1. querySelector `#early-email-form` — bail if not found
    2. Add `submit` event listener with `e.preventDefault()`
    3. Validate email contains `@`
    4. Disable button, change text to "Sending..."
    5. POST FormData via fetch to current page URL (same Netlify Forms AJAX pattern)
    6. On success: hide the form, show `#early-email-success` with `.visible` class
    7. On failure: re-enable button with "Get the Briefing" text, show brief inline error

    **Export both functions** as named exports.

    **CONV-04 note:** Duplicate handling is automatic — Netlify Forms accepts all submissions including duplicates and flags them in the dashboard. No client-side dedup needed. Add a code comment documenting this.

    Create `src/scripts/role-cards.ts` that handles role card selection:

    **initRoleCards() function:**
    1. querySelectorAll `.role-card` — bail if none found
    2. Maintain a Set of selected role strings
    3. On click: toggle role in Set, toggle `.selected` class, toggle `aria-pressed` attribute
    4. Sync Set contents (joined by ", ") into `#selected-roles` hidden input value
    5. Export as named export

    Both files must use TypeScript with proper type annotations on DOM elements (querySelector with `as HTMLFormElement`, etc.).
